Домашнее задание
Создать программу сжатия файлов на основе алгоритма RLE

Цель:
RLEAlgorithm
- Написать функцию сжатия массива по алгоритму RLE - 2 байта

RLETool
- Написать программу для сжатия файлов - 2 байта
- Написать программу для распаковки файлов - 2 байта
При запуске программы без параметров она должна выводить краткую инструкцию,
как её запускать для сжатия/распаковки файлов.

Параметры запуска:
compress input.txt output.rle --basic
decompress output.rle inputD.txt --basic


PackBitsRLEAlgorithm - Реализовать улучшенный алгоритм RLE: сжатие и распаковку - 2 байта
src/test/java/org/micro/kojanni/data_compression_algorithms/rle_test_input.txt
Сравнить работу программы с разными типами файлов:
текст, фото, аудио, zip-архив. 1 байт

Составить отчёт сравнения результата работы двух версий алгоритма с разными файлами. 1 байт

Что наблюдаем: 
Во всех случаях базовый увеличивает размер файла:  Ratio > 1. 
Каждая серия кодируется 2 байтами длины + 1 байт значения, т.е. 3 байта на серию -> Для данных без длинных повторов это неэффективно.

Что касается улучшенного, то тут результаты получше: размер файла либо примерно такой же или выигрываем в размере.
   
Вывод: 
Базовый не применим к реальным данным (текст, медиа, архивы), так как всегда увеличивает размер. 
Эффективен только для искусственных данных с длинными сериями одинаковых байтов. Даже на `rle_test_input.txt` с множеством повторов (OK, A, -) увеличивает размер в 1.54 раза.
Улучшенный вариант дает лучше результат при повторах в тексте(н-р логи).
Уже сжатые форматы (JPG, MP3, ZIP) плохо поддаются RLE-сжатию.
